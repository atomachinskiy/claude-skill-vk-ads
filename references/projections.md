# VK Ads — Projection (прогноз охвата)

## Эндпоинт
`POST /api/v2/ad_plans/<ad_plan_id>/projection.json`

Нужен **существующий ad_plan_id** — можно использовать любой черновик.

---

## Что возвращает

```json
{
  "projection": {
    "impressions_min": 150000,
    "impressions_max": 280000,
    "clicks_min": 1200,
    "clicks_max": 2500,
    "reach_min": 45000,
    "reach_max": 95000,
    "budget_spent_min": 8500,
    "budget_spent_max": 14000
  },
  "predicted_rates": [
    {
      "rate_type": "cpm",
      "value_min": 50,
      "value_max": 120,
      "suggested": 85
    }
  ],
  "histogram": {
    "bins": [
      {"impressions": 10000, "clicks": 80, "budget": 1000},
      {"impressions": 25000, "clicks": 180, "budget": 2500},
      ...
    ]
  },
  "targetings": {
    "geo": {"audience_size": 450000, "impressions_share": 0.35},
    "age_targeting": {"audience_size": 280000, "impressions_share": 0.22}
  }
}
```

### Ключевые метрики
- **impressions_min/max** — диапазон показов при заданном бюджете
- **reach_min/max** — уникальный охват
- **predicted_rates** — рекомендуемые ставки
- **targetings.audience_size** — размер каждого таргета (охват по источнику)
- **histogram** — распределение показов по ставкам

## Тело запроса

```json
{
  "ad_groups": [
    {
      "package_id": 3122,
      "budget_limit": 100000,
      "budget_limit_day": 3300,
      "autobidding_mode": "max_price",
      "max_price": "150",
      "date_start": "2026-05-01",
      "date_end": "2026-05-31",
      "targetings": {
        "geo": {"regions": [5506, 5525]},
        "age_targeting": {"age_from": 25, "age_to": 45},
        "sex": [1],
        "segments": [10001, 10002]
      },
      "banners": [
        {
          "content": {
            "image_1080x1080": {"id": 777888}
          },
          "urls": [{"url": "https://example.com"}]
        }
      ]
    }
  ]
}
```

Можно несколько ad_groups в одном запросе — projection вернёт по каждой.

---

## Когда вызывать ОБЯЗАТЕЛЬНО

1. **Бюджет > 20K₽** — не тратить клиента впустую
2. **Новая ниша / таргет** — впервые идёшь на этот сегмент
3. **Расширение гео** — показать клиенту реальный прирост охвата
4. **Клиент просит запуск** — projection + согласование цифр = сеньор-подход
5. **Смена пакета** (с 3122 на 3127) — ставки и охват другие
6. **Пересборка таргетинга** — сравнить что было vs стало

## Интерпретация (как читать)

### Если impressions_min очень мало (<50K для месячного бюджета)
→ **Таргет слишком узкий**. Пути:
- Расширить гео
- Расширить возраст
- Убрать пол если он не критичен
- Добавить ещё один сегмент в targetings
- Снизить требование (например к сегменту)

### Если CPM prediction выше рынка на 50%+
→ **Слишком много конкурентов в аукционе** на эту аудиторию. Пути:
- Выбрать нишу мельче
- Сменить package (лента → мобилки)
- Или принять и заложить бюджет больше

### Если clicks_max *100 / impressions_max < 0.5% CTR
→ Низкий ожидаемый CTR. Это ОК если креатив продающий — конверсия перевесит. Но если ниша охватная (brand) — сменить таргет.

### Если reach_max < impressions_max / 3
→ Большая частота (пользователь видит 3+ раз). Охват «насытится» быстро.

## Сравнительный анализ двух таргетингов

Запускай projection дважды с разными таргетингами, сравнивай:

```
Таргет А (узкий b2b):
  reach_max = 85K, CPM prediction = 180₽
  
Таргет B (широкий + ретаргет):
  reach_max = 320K, CPM prediction = 95₽

Вывод клиенту: таргет B даёт в 3.8x больше охвата при 2x меньшем CPM.
Рекомендация: идти B, ретаргет добавит точность для конверсий.
```

## Histogram для подбора ставки

Массив `histogram.bins` показывает сколько показов ты получишь при разных ставках:

```
bin 0: 10K показов при ставке 30₽ CPM
bin 1: 25K показов при ставке 50₽ CPM
bin 2: 50K показов при ставке 80₽ CPM
bin 3: 100K показов при ставке 120₽ CPM
bin 4: 180K показов при ставке 180₽ CPM
```

**Находишь колено** — где рост показов начинает стагнировать. Обычно это 2-3 бин. Именно туда ставь `max_price`.

Рекомендация API: `predicted_rates.suggested` — тот же колено по мнению VK.

---

## Audience share по таргетингам

`targetings.<type>.impressions_share` — доля показов которую ДАЁТ конкретный таргет.

Если `segments.impressions_share = 0.05` (5%) — сегмент узкий, основная нагрузка идёт с других таргетингов.

**Паттерн:** если есть ретаргет-сегмент со `share < 0.1` → он «недонабирает» охват, нужен параллельный холодный канал.

## Типовые ошибки projection

| Ошибка | Причина | Решение |
|---|---|---|
| `required_field: package_id` | Не указан пакет | Добавь в ad_group |
| `invalid_targeting` | Некорректный таргетинг | Проверь формат (regions, сегменты) |
| `segment_not_found` | Сегмент из другого кабинета | Перенести / расшарить |
| `projection_failed` | Внутренняя ошибка VK | Retry через 30 сек, потом проверить данные |
| `insufficient_data` | Таргет слишком узкий | Расширь |

## Рецепт: «сеньор-подход» к запуску

```
Шаг 1. Собрать три варианта targetings
  - A: узкий (сегмент + гео + возраст)
  - B: средний (LAL + гео + возраст)
  - C: широкий (гео + возраст, без сегмента)

Шаг 2. Projection по каждому
  cli.py POST /api/v2/ad_plans/<draft>/projection.json --body A.json
  cli.py POST /api/v2/ad_plans/<draft>/projection.json --body B.json
  cli.py POST /api/v2/ad_plans/<draft>/projection.json --body C.json

Шаг 3. Составить таблицу для клиента
  | Вариант | CPM | Reach | Clicks | Бюджет (за мес) |
  | A | 180 | 85K | 900 | 18K |
  | B | 110 | 250K | 2800 | 35K |
  | C | 75 | 500K | 4800 | 50K |

Шаг 4. Рекомендация
  "Начнём с B (оптимальный охват/цена), параллельно A (ретаргет),
  C запустим через месяц когда будет больше сегментов для отсечения
  уже пришедших"

Шаг 5. Согласовать → запуск
```

Это отличает сеньор-таргетолога от джуна.
